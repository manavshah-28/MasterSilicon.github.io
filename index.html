Commands & Setup Guide

Everything you need to simulate, compile, and debug SystemVerilog.

⚡ QuestaSim
Compile SV
vlog -sv name_of_sv_file.sv
Run
vsim -c work.<name of tb> -do "run -all; quit"
Waveform
vsim -voptargs=+acc work.<name_of_testbench>
📝 Testbench Essentials
// VCD waveform dump
initial begin
  $dumpfile("sim.vcd");
  $dumpvars(0, tb);       // dump all signals in scope
end

// Simulation control
$finish;                   // end simulation
$stop;                     // pause (interactive)
#100ns;                    // delay
@(posedge clk);            // wait for rising edge

// Display
$display("val=%0d", val);  // immediate print
$monitor("t=%0t v=%b", $time, val); // auto-print on change

// Assertions
assert(q === 1'b1) else $error("FAIL");
$fatal(1, "TIMEOUT");

How the File System Works

Questions live in questions/<CATEGORY>/<folder-name>/. Each question folder contains four files:

FilePurpose
info.txtTitle, difficulty, tags, related IDs
description.txtConcept explanation (supports Markdown-style headings)
solution.svReference implementation shown with syntax highlighting
testbench.svTestbench to run locally

Step-by-Step: Add a Question

# 1. Copy the template folder
cp -r questions/_TEMPLATE questions/RTL/003-my-new-question

# 2. Edit the four files in the new folder
#    (title, description, solution, testbench)

# 3. Register in manifest.json — add ONE entry:
#    {
#      "id": "RTL-003",
#      "category": "RTL",
#      "folder": "RTL/003-my-new-question"
#    }

# 4. Commit and push — that's it!
💡 The HTML file never needs to change. All content is loaded dynamically from the file system on every page visit.

info.txt Format

title: Your Question Title
difficulty: Easy        # Easy | Medium | Hard
tags: tag1, tag2, tag3
related: RTL-001, DFT-001

description.txt Formatting

Use Markdown-style headings and special callouts:

## Section Heading (renders as teal uppercase header)
### Sub-heading

Regular paragraph text here.

`inline code` renders with teal highlight.

> TIP: Prefix with "> TIP:" for a green callout box.
> WARNING: Prefix with "> WARNING:" for an amber callout box.
